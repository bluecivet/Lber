about the project

the project is a mini Uber.
People have to sign in to an account first in order use the web.
People can sign in at main pages (most html file expect driverPanel.html and userPanel.html).
the main page is just talk about the web
tutor.html are instructions that tach users how to use the page.

some function of the web

costomer can place their order and driver can accept orders in their control panel.
costomer have to have enough point holding in order to place orders.
once costomer place an order, drivers can see it and decide they should accept or not. of course, if other diver accept it first, then other driver can not accept any more.

if driver finish delivery the order, costomers have to click finish button in current order in their panel. This means the order is end.
point will transfer from costomer to driver.

common function

both user and driver have some common function they can use
they can view their order history in their panel
they can see their current order
there is informaton in order that say the order is accepted or not in both order history and current history.
they are able to change their personal setting like user name and password
finally, they can add point by their own.

form input error handling

I use h5 and javascript to handle error.
there are low level of api: formValidationAPI.js
the api is for checking h5 error like require pattern ...
they can be commonly use
some other type of error, people may need to write it out
some example use in loginCheck.js and signinCheck.js
panelFormErrorHandle.js is better stucture a more common and easy to use
I make this to an object just new the object, hide other logical code
all of them, you have to create a errorMessage object for show which type of error and show the message out.

form submitting

although, there are action attrubute in html form, I use ajax to submit form. I will check form again in server side like php/form/formUtil.php
if the form are successful submit, server will send a "good" back.
if some errors, server will send just the error message or a json object that include where should place the error and error message
